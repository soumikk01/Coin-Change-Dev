üí° The Coin Change Problem

Given a target amount and a set of coin denominations, find the minimum number of coins needed to make that amount.

‚ö° Greedy Algorithm

How it works: Always picks the largest coin that doesn't exceed the remaining amount.

  • Fast: Very quick - O(n log n) time complexity
  • Simple: Easy to understand and implement
  • Not Always Optimal: Doesn't guarantee minimum coins for all coin systems
  • Example: For amount 6 with coins {1, 3, 4} ‚Üí Uses {4, 1, 1} = 3 coins (not optimal!)

üéØ Dynamic Programming (DP)

How it works: Builds solutions from bottom-up, trying all possibilities to find the true minimum.

  • Always Optimal: Guarantees minimum coins for ANY coin system
  • Slower: O(amount √ó coins) time complexity
  • More Complex: Uses memorization to avoid redundant calculations
  • Example: For amount 6 with coins {1, 3, 4} ‚Üí Uses {3, 3} = 2 coins ‚úì (optimal!)

‚öñÔ∏è When to Use Each

Use Greedy When:
  • Speed is critical
  • Using standard coin systems (US, EUR, INR)
  • Approximate solution is acceptable
Use DP When:
  • Need guaranteed optimal solution
  • Using custom/unusual coin systems
  • Accuracy is more important than speed
